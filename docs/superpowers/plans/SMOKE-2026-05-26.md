## Known risks worth eyeballing during smoke

1. **JSS v0.0.200+ availability:** flake uses `bunx --bun javascript-solid-server@latest`. If npm lags or auth fails, jss process won't start. Mitigation: try `npx -y javascript-solid-server@latest start --mcp ...` first to verify network/install.

2. **CC HTTP MCP transport:** `.mcp.json` uses `type: "http"`. If CC version requires different syntax, MCP server won't register. Run `claude` once and inspect `/mcp` status output.

3. **WS-subscribe path encoding:** `PodClient.subscribe` sends `sub <full-url>` and parses `pub <full-url>`. JSS solid-0.1 protocol should match this exactly. If event paths come back with trailing slashes or query strings, the `localPath` extraction in rdf.ts:subscribePodChanges вЖТ reloadResource may 404.

4. **Oxigraph WASM size:** dist build shows wasm at 4MB. First-load latency only.

5. **process-compose readiness probe:** `jss` health-check uses HTTP GET `/`. JSS may not serve a 200 on root in single-user mode вАФ adjust probe path if startup hangs.
